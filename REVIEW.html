Summary

When a GitHub org invitation fails (e.g. 2FA not configured), the stale failed-invitation record blocks subsequent re-invites. This PR makes peribolos detect those records and delete them before issuing a fresh invite.

Control Flow

DeleteOrgInvitation accepts both 204 and 404 exit codes — if the invitation is already gone, no error. Correct semantics.

Items to Discuss During Review

Note time.Time and omitempty
types.go:1201 — FailedAt time.Time `json:"failed_at,omitempty"`

Go's encoding/json does not treat the zero time.Time as "empty" — only JSON null is omitted. So omitempty here is inert. Not a functional problem because peribolos only reads from GitHub (never serializes OrgInvitation back), but it might mislead future readers into thinking zero times will be suppressed.

Consider: use *time.Time if omitempty behavior is intended, or drop the omitempty tag.

Note Whitespace-only reformatting
main.go:47-64, main_test.go:1866-1870

The options struct and TestDumpOrgConfig struct get alignment-only changes that add noise to the diff. Not wrong, but if the project prefers clean diffs, these could be a separate commit or omitted.

Note intSliceToString uses lexicographic sort
main_test.go:849-855

Converts ints to strings then calls sort.Strings. Lexicographic order means "10" < "2". This is harmless for the small test values used, but sort.Ints on the original slices would be more robust. Very minor.

Info OrgInvitation.ID is now populated for pending invitations too
types.go:1198

Adding the ID field is backwards-compatible and purely additive. Existing code that calls ListOrgInvitations will now also receive the ID, which could be useful for future features (e.g. revoking specific pending invitations). No action needed, just context.

Advisor Suggestions (non-blocking)

Suggestion Per-invitation logging noise

Consider logging a single summary line (e.g., "Found N failed invitations for org X") instead of one Info line per failed invitation. For orgs with many stale invitations this could be noisy, and the existing orgInvitations path does not log individual items.

Suggestion GHE compatibility note in PR description

The /failed_invitations endpoint may not be available on all GitHub Enterprise Server versions. Operators not using --ignore-invitees on older GHE could see org sync abort. A one-line note in the PR description would help operators evaluate impact before upgrading.

Suggestion time.Time omitempty tag

The omitempty on FailedAt time.Time is inert with encoding/json since zero time.Time is not considered empty. Cosmetic only since peribolos only deserializes, but worth knowing if the struct is ever used for serialization.
