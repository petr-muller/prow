Overview

Moves the standalone transfer-issue plugin into the consolidated issue-management plugin. The pkg/plugins/transfer-issue/ directory is removed. Its logic is absorbed into the issue-management package, and the routing/validation is extracted into the handleIssues dispatcher, following the same pattern used for link-issue and pin-issue.

Deployment Risk

Risk Silent functionality loss for operators

The plugin handler registration changes from "transfer-issue" to "issue-management". Operators who have transfer-issue in their plugins.yaml but not issue-management will silently lose the /transfer-issue command — no error, no log, no warning.

  • Operators who already have issue-management enabled get transfer support automatically.
  • No config schema, YAML/JSON tags, or CLI flags are changed.
  • Rollback is safe — no data changes involved.
