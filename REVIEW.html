Recommendation: REQUEST_CHANGES

All three independent reviewers flagged the XSS vulnerability and the lack of tests. The XSS issue is specific to this lens: unlike the html lens which renders inside an iframe sandbox, the summary lens injects directly into the Deck page DOM, making it a real injection vector. Combined with zero test coverage in a codebase where every comparable lens has tests, these two issues warrant changes before merge.

Reviewer Perspectives

1/3 — Code Quality Request Changes
  • XSS via unsanitized Markdown (critical): blackfriday.Run() output cast to template.HTML bypasses auto-escaping. Unlike the html lens (iframe sandbox), this renders directly in the Deck DOM — scripts in artifacts execute in the application context.
  • No tests: Zero coverage. Every comparable lens (html, buildlog, junit, metadata, podinfo, links) has tests.
  • Misleading error message: Line 98 logs "Error executing template." but the failure is template parsing.
  • Silent empty result: If all artifacts fail to read, an empty body renders with no indication of failure.
  • Positive: Follows established lens patterns well, good structured logging, correct spyglass.contentUpdated() integration.
2/3 — Maintainability Request Changes Burden: Medium
  • Unmaintained dependency: blackfriday/v2 has not had a release since April 2021. The Go ecosystem has moved to goldmark. Adopting an unmaintained library means future bugs or CVEs require a fork or migration.
  • Unsanitized HTML as maintenance debt: When eventually flagged as a security issue, fixing it will change rendering behavior — a disruptive patch. Better to address upfront.
  • No tests: Adding them now is far cheaper than later when the author's context is gone.
  • Positive: Follows exact html lens pattern. Cleanly scoped — one package, one import, one dependency. Easy to revert. Good structured logging.
3/3 — Deployment Risk Risk: Low
  • Entirely additive: No existing code paths, configs, or behavior modified.
  • Opt-in activation: Lens only activates when operators explicitly configure it in Spyglass config. Without configuration, it never renders.
  • Safe rollback: Removing the lens is trivial — leftover config becomes a no-op, not an error.
  • No config migration needed, no new permissions, no behavioral changes to existing lenses.
  • XSS surface noted: Artifact content is from CI outputs (narrower threat model), but still renders in the Deck DOM context.

Required 3/3 reviewers XSS via unsanitized Markdown rendering

pkg/spyglass/lenses/summary/summary.go:88-92

blackfriday.Run() converts Markdown to HTML but does not sanitize the output. Markdown can embed raw HTML including <script> tags and event handlers. Wrapping the result in template.HTML bypasses Go template auto-escaping. Unlike the html lens (which uses an iframe sandbox via srcdoc), this lens injects directly into the Deck page DOM — scripts in a .md artifact would execute in the application context.

Suggested fix

import "github.com/microcosm-cc/bluemonday"

p := bluemonday.UGCPolicy()
safeHTML := p.SanitizeBytes(blackfriday.Run(content))

Alternatives: use blackfriday's SkipHTML extension flag, render in a sandboxed iframe like the html lens, or use goldmark with raw HTML disabled by default.
