Root Cause

Prow combines two Prometheus registries: prometheus.DefaultRegistry (Prow's own metrics + auto-registered Go/process collectors) and ctrlruntimemetrics.Registry (controller-runtime metrics + its own Go/process collectors). To avoid duplicates, Prow unregisters Go/process collectors from controller-runtime's registry by creating equivalent instances and calling Unregister(), which matches by descriptor set.

The dependency bump changed controller-runtime's collector registration:

Dependency Before (working) After (broken)
prometheus/client_golang v1.20.5 v1.22.0
controller-runtime v0.20.1 v0.21.0
Go collector init collectors.NewGoCollector() collectors.NewGoCollector(collectors.WithGoCollectorRuntimeMetrics(collectors.MetricsAll))

The MetricsAll option produces a different descriptor set. Prow's unregister call (using prometheus.NewGoCollector() with default options) no longer matches, fails silently, and both registries serve go_* metrics — causing 29 "collected before with the same name and label values" errors.

prometheus.DefaultRegistry ctrlruntimemetrics.Registry init(): NewGoCollector() init(): NewGoCollector(MetricsAll) init(): NewProcessCollector() init(): NewProcessCollector() + Prow custom metrics + controller-runtime reconcile metrics | | | Prow tries to Unregister: | | prometheus.NewGoCollector() != collectors.NewGoCollector(MetricsAll) | MISMATCH → Unregister fails silently | | +----------- Gatherers{...} ------------+ | DUPLICATE go_* metrics → ERROR

Fix Location

File: pkg/metrics/metrics.go, lines 53–56

// Current (broken):
//nolint:staticcheck
ctrlruntimemetrics.Registry.Unregister(prometheus.NewGoCollector())
//nolint:staticcheck
ctrlruntimemetrics.Registry.Unregister(prometheus.NewProcessCollector(prometheus.ProcessCollectorOpts{}))
// Fixed:
ctrlruntimemetrics.Registry.Unregister(collectors.NewGoCollector(
    collectors.WithGoCollectorRuntimeMetrics(collectors.MetricsAll)))
ctrlruntimemetrics.Registry.Unregister(collectors.NewProcessCollector(
    collectors.ProcessCollectorOpts{}))

Also add import: "github.com/prometheus/client_golang/prometheus/collectors"

Solution Approaches

Approach 1: Match controller-runtime's collector options (Recommended)

Update unregister calls to use collectors.NewGoCollector(collectors.WithGoCollectorRuntimeMetrics(collectors.MetricsAll))

+ Minimal change, 1 file, ~5 lines, directly fixes the mismatch

- Still fragile — will break if controller-runtime changes options again

Approach 4: Unregister from DefaultRegistry instead

Remove Go/process collectors from prometheus.DefaultRegistry (where they're guaranteed to match) and keep controller-runtime's richer MetricsAll set.

+ More robust long-term, keeps richer metrics

- Slightly different metric set exposed (more runtime metrics)
